# 人格蒸馏指南

把一个人的原始素材（推文、访谈、播客、文章、聊天记录）变成结构化的 AI 人格。

[English](distillation-guide.md) | 中文

## 总览

```
原始素材 → 采集 → 3-Pass 蒸馏 → 结构化输出 → Skill
```

成本估算：蒸馏 1 万条推文 + 50 小时访谈 ≈ $5–15 API 费用。

***

## 第一步：采集原始素材

尽可能多地收集。输入量越大，蒸馏质量越高。

### 数据源类型

#### 社交媒体（高价值 — 暴露真实声音）

| 平台          | 采集方式                                                    | 推荐工具                              | 注意事项                          |
| ----------- | ------------------------------------------------------- | --------------------------------- | ----------------------------- |
| X / Twitter | 账号数据导出（Settings → Your Account → Download Archive）或 API | Twitter Archive, Tweepy, X API v2 | 导出包含所有推文（含回复）。回复往往比主贴更能暴露人格。  |
| 即刻          | 手动导出或爬取                                                 | 浏览器自动化, Jike API（非官方）             | 即刻动态是金矿——短、随意、真实。优先用于「内部语录」层。 |
| LinkedIn    | 手动复制或数据导出                                               | LinkedIn 数据导出                     | 正式声音——适合校准「写作模式」，对日常人格用处较小。   |
| 微博          | 微博数据导出或爬取                                               | 微博 API, 浏览器自动化                    | 过滤掉没有评论的纯转发。                  |
| Reddit / 论坛 | 用户主页爬取                                                  | Reddit API, Pushshift             | 评论比帖子人格信号更强。                  |

**目标量**：200+ 条起步，500+ 条理想。

#### 长文写作（高价值 — 暴露思维结构）

| 来源                | 采集方式                                   | 推荐工具                                      | 注意事项                                         |
| ----------------- | -------------------------------------- | ----------------------------------------- | -------------------------------------------- |
| 博客 / Newsletter   | 复制或爬取                                  | Readability 解析器, Jina Reader, Markdownify | 保留原始结构（标题、列表）——它暴露了思维组织方式。                   |
| 微信公众号             | URL 提取 + 转换                            | 公众号爬虫, 浏览器自动化                             | 微信反爬严格，可能需要手动复制。保存完整 HTML 以获取格式信号。           |
| Substack / Medium | RSS 订阅或直接爬取                            | RSS 解析器, Jina Reader                      | Substack 有干净的 RSS。Medium 需要绕过鉴权。             |
| 内部文档（需授权）         | 从 Notion / Google Docs / Confluence 导出 | 平台导出工具                                    | 内部文档是最高信号源——展示了一个人在没有「表演」时怎么思考。**必须获得明确授权。** |

**目标量**：10+ 篇起步，30+ 篇理想。

#### 访谈 & 播客（极高价值 — 暴露即时思维）

| 来源         | 采集方式                | 推荐工具                                         | 注意事项                                                                              |
| ---------- | ------------------- | -------------------------------------------- | --------------------------------------------------------------------------------- |
| 播客节目       | 下载音频 → 转录           | yt-dlp（支持多平台）, Whisper, Deepgram, AssemblyAI | **说话人分离（Speaker Diarization）是关键**——必须区分哪些话是 TA 说的。推荐 Whisper large-v3 + pyannote。 |
| YouTube 访谈 | 字幕提取或下载音频 → 转录      | yt-dlp, YouTube Transcript API               | YouTube 自动字幕准确率 \~90%。有人工字幕优先用人工的。                                                |
| 小宇宙播客      | 下载音频 → 转录           | 浏览器自动化, Whisper                              | 中文播客转录推荐 Whisper large-v3 或 FunASR，中文准确率更高。                                       |
| 会议演讲       | YouTube / 活动平台 → 转录 | yt-dlp, Whisper                              | 演讲暴露「展示模式」——对校准正式写作层有用。                                                           |
| 媒体采访（文字）   | 从出版物复制              | 网页爬取                                         | 文字采访经常编辑掉了人格信号。音频/视频永远更真实。                                                        |

**目标量**：3+ 小时音频起步，10+ 小时理想。

#### 书籍 & 长文档

| 来源            | 采集方式            | 推荐工具                         | 注意事项                              |
| ------------- | --------------- | ---------------------------- | --------------------------------- |
| 出版书籍          | PDF/EPUB → 文本提取 | PyMuPDF, Calibre, pdfplumber | 重点提取表达观点的段落，而非引用数据的段落。            |
| PPT / 演示文稿    | PDF/PPTX → 文本提取 | python-pptx, PyMuPDF         | PPT 展示了 TA 的信息层级——什么内容放在最前面。      |
| 邮件 Newsletter | 归档爬取或转发收集       | -                            | Newsletter 的声音往往最接近 TA 的「真实写作声音」。 |

#### 聊天记录（最高真实度 — 需授权）

| 来源           | 采集方式                   | 推荐工具                | 注意事项                                   |
| ------------ | ---------------------- | ------------------- | -------------------------------------- |
| Slack        | Workspace 导出（管理员）或频道导出 | Slack Export        | 过滤到只保留 TA 的消息。包含 thread 回复——人格信号集中在那里。 |
| Discord      | Bot 导出                 | DiscordChatExporter | 同 Slack——thread 回复 > 顶层消息。             |
| 微信 / 私聊（需授权） | 手动截图 → OCR，或聊天记录导出     | 微信备份工具, OCR         | 最真实的数据源。即使只有 50 条消息，也能显著提升日常聊天模式的质量。   |
| 会议纪要         | Zoom/腾讯会议/飞书录制 → 转录    | 平台自带转录, Whisper     | 会议行为（打断、追问、总结）是强人格信号。                  |

**重要**：使用私人通信记录前，必须获得明确同意。

### 语料质量自检清单

进入蒸馏前，确认语料覆盖以下维度：

* [ ] **日常声音** — 至少 50 条聊天/随意风格的数据点

* [ ] **正式声音** — 至少 5 篇长文

* [ ] **即时思维** — 至少 1 小时访谈/播客

* [ ] **自我反思** — 至少几条承认错误或不确定性的记录

* [ ] **领域专长** — 覆盖 TA 的核心话题

* [ ] **矛盾 / 演变** — 至少一个 TA 改变过看法的例子

**最低语料量**：50+ 数据点可用，200+ 可信，500+ 接近真人。

### 原始语料目录结构

将所有采集的素材存放在 `_raw/` 目录：

```
_raw/
├── social/
│   ├── x-tweets-export.json
│   └── jike-posts.json
├── writing/
│   ├── blog-post-1.md
│   └── blog-post-2.md
├── interviews/
│   ├── podcast-ep42-transcript.md
│   └── youtube-interview-transcript.md
├── chat/
│   └── slack-messages.json
└── sources-index.md          # 采集了什么、从哪来、什么时候
```

## 第二步：蒸馏（3-Pass Pipeline）

### Pass 1：分块 + 打标

将所有原始文本切成块（每块 \~2000 字），用 AI 为每块提取标签：

```yaml
topic: ["管理", "产品策略"]
type: "观点表达"        # 观点 / 事实陈述 / 决策 / 情绪 / 闲聊
stance: "反对微观管理"
quote_worthy: true      # 是否有金句
time_period: "2024-Q3"
```

**Prompt 模板：**

```
分析以下 {person_name} 的文本。提取：
1. 讨论的主要话题（关键词列表）
2. 内容类型：观点、事实陈述、决策、情绪表达、还是闲聊
3. TA 的立场（一句话）
4. 是否包含值得保留的金句（true/false）
5. 可识别的时间段

文本：
{chunk}
```

这一步用便宜快速的模型（Haiku / GPT-4o-mini）跑。它是分类任务，不是生成任务。

### Pass 2：聚合 + 去重 + 排序

按话题聚合所有块，然后对每个话题：

1. **去重**：移除语义相似的块（只保留表达最清晰的版本）
2. **排序**：按 quote\_worthy + 互动数 + 时效性打分
3. **截断**：每个话题保留 top 20–50 条

```python
# 伪代码
topics = group_by(all_chunks, key="topic")

for topic in topics:
    deduplicated = semantic_dedup(topic.chunks, threshold=0.85)
    ranked = rank(deduplicated, by=["quote_worthy", "engagement", "recency"])
    topic.selected = ranked[:50]
```

### Pass 3：结构化写作

将精选素材交给强模型（Opus / Sonnet / GPT-4），按以下 Prompt 分别生成各输出文档。

#### 生成 `_persona/rules.md`：

```
你是一个人格架构师。根据以下关于 {person} 的原始素材，
提取 TA 的人格特征并写成结构化文档。

必须包含的章节：
1. 身份定位（当前角色、背景、公开身份）
2. 核心人格（5-8 个核心特质，附证据）
3. 思维框架（TA 怎么拆解问题——具体的、有名字的框架）
4. 决策风格（怎么做选择、优先考虑什么）
5. 口头禅（高频表达，附使用语境）
6. 绝对不会做的事（TA 永远不会说的话或做的事）

原始素材：
{selected_chunks_personality}
```

#### 生成 `_persona/communication.md`：

```
分析以下样本中 {person} 的沟通模式。
极其具体地记录 TA 的语言风格。

必须包含的章节：
1. 模式检测（什么场景触发什么沟通风格）
2. 日常模式规则（句子长度、用词、标点习惯、emoji 使用）
3. 正式模式规则（结构、修辞手法、标志性技巧）
4. 语言混用规则（什么时候切换语言、哪些术语始终保持原文）
5. 真实句子样本（每种模式 10+ 条真实引用）
6. 速查规则表

原始素材：
{selected_chunks_communication}
```

#### 生成 `_persona/values.md`：

```
根据以下素材，按确信程度排列 {person} 的核心信念。

结构：
- Tier 1：深度确信（从不动摇）— 3-5 个
- Tier 2：强烈倾向（高度自信，保留修正空间）— 5-7 个
- Tier 3：活跃探索（有假设，待验证）— 3-5 个

每个信念需要：
- 一句话概括
- 支撑引用（附出处）
- 确信程度的证据

原始素材：
{selected_chunks_values}
```

#### 生成 `_quotes/iconic.md`：

```
从以下素材中，选出 {person} 最有代表性的 30 条金句。
按主题分组。每条附准确出处和日期。

筛选标准：
- 能反映 TA 的思维方式或价值观
- 令人印象深刻、可引用
- 有代表性（不是偶然的极端言论）

原始素材：
{all_quote_worthy_chunks}
```

#### 生成 `_quotes/internal.md`：

```
选出 20-30 条 {person} 私下坦诚的非正式发言——
TA 在非公开场合才会说的那种话。

这些发言应该展示：
- 不在「表演」时的说话方式
- 幽默和自嘲
- 怎么布置任务或给反馈
- 即时反应

原始素材：
{selected_chunks_casual}
```

#### 生成每个 `_knowledge/{topic}.md`：

```
写一篇关于 {person} 对 {topic} 看法的结构化知识文档。

章节：
1. 核心观点（3-5 个关键立场，每个附支撑引用）
2. 立场演变（如果 TA 的看法随时间变化）
3. 决策逻辑（为什么持这些观点——TA 的推理链）
4. 常见误解（外界经常误读 TA 立场的点）

原始素材：
{selected_chunks_for_topic}
```

## 第三步：审核 & 校准

生成完成后，做一轮校准：

1. **声音校验**：大声朗读人格文档。听起来像 TA 吗？
2. **边界校验**：「绝对不做」的规则准确吗？测试边界案例。
3. **知识校验**：知识文档事实准确吗？删掉不确定的内容。
4. **引用校验**：所有金句都是真实的、归属正确的吗？
5. **模式校验**：分别测试日常聊天和正式写作模式。切换自然吗？

