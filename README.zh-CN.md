# soul.skill

**从公开素材中蒸馏一个人的思维方式、语言模式与判断框架。**

我们对一个人的了解，大多停留在一种奇特的中间状态——读过 TA 所有的文章，听过 TA 每一期播客，在社交媒体上追踪 TA 的每一次表态，却仍然无法回答一个简单的问题：如果此刻把一个全新的话题丢给 TA，TA 会怎么想？

不是 TA 说过什么。是 TA **会**说什么。

这个"会"字背后，是观点之下的判断框架，是语言习惯深处的思维节奏，是 TA 在某些话题上反复修正却从未公开承认的立场演变。这些东西散落在几千条推文、几十小时的播客、几十篇长文之中——它们从未被系统性地整理过。

soul.skill 做的事情，某种程度上，是一种数字考古：把散落的碎片重新拼合，直到一个人的思维轮廓从素材中浮现出来。

***

## 它做什么

soul.skill 把散落在各处的素材——播客、推文、文章、聊天记录——系统性地蒸馏成一个结构化的 AI 人格。具体来说：

- **基于素材，不是基于印象**：AI 实际读完所有素材，而非依赖训练数据中的模糊记忆
- **观点可溯源**：每个判断挂着原始引用，而非凭空生成
- **记录立场演变**：同一话题在不同时期的表态与转向都会被保留
- **标注边界**：素材中没有的信息，标注为未知，不编造

***

## 素材来源

并非所有素材的价值相等。一个人在即兴对话中暴露的判断框架，远比一篇精心打磨的文章更接近 TA 的真实思维。

| 来源       | 采集方式                | 信号密度  | 说明                                                                             |
| -------- | ------------------- | ----- | ------------------------------------------------------------------------------ |
| 播客与访谈    | 音频转录 / YouTube 字幕   | ★★★★★ | 即兴反应最难伪装——这是最接近"原矿"的素材                                                         |
| 个人博客与公众号 | 全文抓取                | ★★★★★ | TA 亲手写的内容，每个用词都是一次选择                                                           |
| 聊天记录     | 手动导出 / 截图           | ★★★★★ | 最私人的语言样本——语气、口头禅、情绪反应的密度远超公开写作                                                 |
| 推文与社交动态  | 自助导出（本人配合）/ 即刻等平台采集 | ★★★★☆ | 日常语言的大样本，量的优势可以弥补单条的浅。注意：X/Twitter 目前无法免费自动采集他人推文，需本人导出数据包配合；如不可得，建议用即刻等替代来源补偿 |
| 演讲与发布会   | 字幕提取                | ★★★☆☆ | 有表演成分，信号经过了一层修饰                                                                |
| 新闻报道     | RSS / 搜索抓取          | ★★☆☆☆ | 经过记者转述和编辑加工，人格信号较弱                                                             |

## 使用

```
/create-soul
```

输入人物名称，提供素材链接或上传文件。蒸馏过程自动执行。

完成后，用 `/{slug}` 与蒸馏出的思维模型对话。

***

## 效果

> 素材：某 AI 创业者，50 期播客 + 800 条即刻动态 + 30 篇公众号文章

**请教**

```
用户       ❯ 我在考虑做一个 AI 写作工具，你觉得这个方向怎么样？

soul.skill ❯ 写作工具这个品类，核心矛盾不在技术，在定义。
             你是在帮人写得更快，还是帮人想得更清楚？
             前者的终局是 ChatGPT 加一个输入框——你不可能
             比它更快。后者其实不是写作工具，是思维工具。
             这两个东西的产品逻辑完全不一样。
```

**边界**

```
用户       ❯ 能不能帮我写一段融资 pitch？

soul.skill ❯ 我不太擅长写那种结构化的 pitch。"万亿市场、
             三年千万 ARR"——这类话术我说不出来。我跟
             投资人聊，一般就是讲我在解决什么问题，为什么
             是现在。你要不先跟我说说你在做什么。
```

## 生成结构

蒸馏完成后，产出以下目录：

```
{person}-soul/
├── _persona/
│   ├── rules.md              # 身份、信条、决策风格、禁忌
│   ├── communication.md      # 多模式语言系统——正式写作 / 口语 / 社交媒体
│   └── values.md             # 价值观排序与冲突时的取舍
│
├── _knowledge/
│   ├── {topic}.md            # 核心观点 + 原始引用 + 立场演变 + 决策逻辑
│   └── ...
│
├── _quotes/
│   ├── iconic.md             # 按主题索引的代表性表达
│   └── internal.md           # 非正式场合的真实语言样本
│
└── _meta/
    └── sources.md            # 素材来源与覆盖率
```

## 使用场景

* "我在纠结要不要全职创业，你怎么看？"——用 TA 的判断框架分析你的处境

* "帮我用你的风格写一段关于 AI Agent 的评论"——还原 TA 的语言节奏与用词偏好

* "你 2022 年说过纯视觉路线是对的，现在还这么认为吗？"——追踪立场演变

* "如果明天 OpenAI 发布了 GPT-6，你第一反应是什么？"——基于已有框架推演

* "最近在看什么书？"——从素材中提取阅读偏好与推荐逻辑

***

## 注意事项

素材的质量比数量更重要。3 小时的播客逐字稿可能比 3000 条转发更有价值——关键是素材中包含多少真实的判断、情绪和即兴反应。蒸馏方法同样关键：同样的素材，逐篇深读和批量扫描的产出质量完全不同。

这是素材的蒸馏，不是读心术。素材中不存在的信息，不会被发明出来。你永远只能复刻这一秒之前的 TA。
